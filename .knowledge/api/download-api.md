---
id: api:download-api
type: api
title: File Access API
---
File Access API is separate from api:upload-api and exposes file content plus future derived retrieval.

```yaml
base_paths:
  upload_api: /api/upload
  frontend_file_api: /api/file
  frontend_files_api: /api/files
  backend_file_api: /internal/file
path_rules:
  key: object key path encoded for URL path use
  keys: comma-separated URL path escaped object keys for archive download
  content_suffix: /content returns original file bytes without forced attachment
  download_suffix: /download returns original file bytes with attachment Content-Disposition
  comma_separator:
    - comma separates keys only in /api/files/{keys}
    - literal comma inside object key must be percent-encoded
  reserved_segments:
    - shared
  reason:
    - keep upload mutation routes isolated
    - leave room for thumbnails, previews, variants, and metadata
endpoints:
  get_content_proxy:
    method: GET
    path: /api/file/{key}/content
    config_gate: allow_frontend_file_access
    delivery_mode: api_proxy_download
    response:
      content_disposition: absent unless source metadata requires otherwise
    behavior:
      - reject unless policy:object-access-policy allows api_proxy_download
      - stream original object bytes through streamuploader frontend API
      - allow browser inline rendering for images, PDF, text, and other renderable types
      - do not force attachment download
      - hide bucket endpoint and object storage credentials
      - support Range and If-Range when enabled
      - emit audit event
  get_download_proxy:
    method: GET
    path: /api/file/{key}/download
    config_gate: allow_frontend_file_access
    delivery_mode: api_proxy_download
    response:
      content_disposition: attachment with file name
    behavior:
      - same object streaming rules as get_content_proxy
      - force browser download when user navigates to endpoint
  get_shared_content:
    method: GET
    path: /api/file/shared/{shared_key}/content
    config_gate: enable_shared_key and allow_frontend_file_access
    delivery_mode: shared_key_proxy_download
    response:
      content_disposition: absent unless source metadata requires otherwise
    behavior:
      - resolve api:shared-key-api
      - stream target object through streamuploader frontend API
      - allow browser inline rendering
      - do not force attachment download
      - deny expired, revoked, or malformed shared key records
  get_shared_download:
    method: GET
    path: /api/file/shared/{shared_key}/download
    config_gate: enable_shared_key and allow_frontend_file_access
    delivery_mode: shared_key_proxy_download
    response:
      content_disposition: attachment with file name
    behavior:
      - resolve api:shared-key-api
      - stream target object through streamuploader frontend API
      - force browser download when user navigates to endpoint
      - deny expired, revoked, or malformed shared key records
  get_zip_archive:
    method: GET
    path: /api/files/{keys}
    config_gate: allow_frontend_file_access
    delivery_mode: api_proxy_download
    query:
      filename: optional archive file name, default files.zip
    response:
      content_type: application/zip
      content_disposition: attachment archive name
    behavior:
      - parse keys as comma-separated encoded object keys
      - authorize every key with same frontend file access policy
      - stream zip archive without staging full archive when possible
      - preserve each original file name when metadata is available
      - reject request when key count or estimated bytes exceed archive policy limits
      - emit one archive audit event plus per-object entries when configured
  create_presigned_url:
    method: POST
    path: /internal/file/presigned-url
    delivery_mode: s3_presigned_download
    body:
      object_key: required S3 object key
      file_name: optional download name
      ttl_seconds: optional bounded TTL
    response:
      url: S3 presigned GET URL
      expires_at: timestamp
    behavior:
      - backend API asks S3 for presigned GET URL
      - caller returns or redirects to URL
      - bucket can stay private
  get_thumbnail:
    method: GET
    path: /api/file/{key}/thumbnail/{preset}
    config_gate: allow_frontend_file_access
    status: future
    behavior:
      - resolve generated thumbnail data:derived-asset
      - return placeholder or generated bytes according to api:derived-asset-serving-api
  get_variant:
    method: GET
    path: /api/file/{key}/variants/{variant_key}/content
    config_gate: allow_frontend_file_access
    status: future
    behavior:
      - resolve generated or transformed data:derived-asset
      - stream bytes or return readiness placeholder
range_support:
  - pass through byte ranges to S3
  - return 206 Partial Content
  - preserve Content-Type, Content-Encoding, ETag, and Content-Range
archive_support:
  - range requests do not apply to generated zip archive
  - archive generation should stream from S3 objects into zip writer
  - duplicate archive entry names must be disambiguated deterministically
demo_modes:
  - direct_public_bucket bypasses streamuploader and is implemented by demo app
  - s3_presigned_download
  - api_proxy_download
  - shared_key_proxy_download
non_api_modes:
  direct_public_bucket:
    owner: demo app
    behavior:
      - demo app builds public S3 or CDN URL from configured public bucket endpoint and object_key
      - browser accesses S3 or CDN directly
      - streamuploader is not called for URL creation or byte serving
references:
  - policy:object-access-policy
  - policy:shared-key-policy
  - api:shared-key-api
  - api:derived-asset-serving-api
  - policy:audit-log-policy
```
